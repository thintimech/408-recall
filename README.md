# 408 Recall

408 Recall 是一个面向计算机考研 408 的本地优先主动回忆工具。它帮助用户围绕数据结构、计算机组成原理、操作系统、计算机网络创建记忆卡片，并通过间隔复习算法安排每日复习队列。

## 功能概览

### 核心复习闭环

- 内置 408 四门课基础章节树，支持自定义知识点节点
- 9 种卡片类型：概念卡、对比卡、填空卡、流程卡、公式卡、易错卡、大题模板卡、题型套路卡、定理条件卡
- 主动回忆复习：先看正面，再显示答案，四级评价（忘了/困难/记得/熟练）
- SM-2 变体间隔复习算法，自动计算下次复习日期
- 支持跳过、撤销、本轮复习总结
- 键盘快捷键：空格翻面、1234 评分、Z 撤销、S 跳过
- 考前突击模式：按科目集中复习所有卡片，打乱顺序，不受间隔调度限制

### 富文本渲染

- 卡片正面/背面支持 Markdown 渲染
- LaTeX 公式（行内 `$...$`，块级 `$$...$$`）
- 代码块、表格、列表、引用等完整 Markdown 语法
- 编辑时实时预览 + 快捷插入工具栏

### 学习数据可视化

- 过去 90 天复习热力图
- 四门课掌握率进度条
- 每日复习量柱状图
- 高遗忘卡片和长期未复习卡片识别

### AI 辅助

- 接入 OpenAI 兼容 API（默认 DeepSeek），支持多提供商切换
- 9 种专用 Prompt 模板，流式批量生成卡片，逐张预览确认
- AI 答案补全：根据问题自动生成答案
- AI 答案审查：检查答案的正确性、错误和遗漏
- AI 深度追问：复习评分后生成变式问题，测试更深层理解
- AI 遗忘分析：对反复遗忘的卡片分析原因并给出改进建议
- AI 学习诊断：基于复习数据分析薄弱环节和学习趋势
- AI 上下文对话：复习时可就当前卡片内容多轮追问
- API Key 仅存储在本地 IndexedDB，不离开浏览器

### 云同步

- 基于 Supabase（PostgreSQL + Auth）实现跨设备数据同步
- 本地优先：IndexedDB 仍是主存储，离线完全可用
- 增量同步：只推送自上次同步后有变更的数据
- Last-write-wins 冲突策略，以 `updatedAt` 时间戳判断，本地更新不会被旧的云端数据覆盖
- 复习记录追加型同步，天然无冲突
- 账号注册/登录/退出，API Key 不参与同步

### 理解笔记（Insights）

- 记录学习过程中的个人顿悟和理解
- 可关联知识点，也可独立存在
- 支持 Markdown 格式书写
- 独立浏览页面，支持搜索
- 与移动端双向同步



- 全局快捷添加卡片（N 键或顶栏按钮，弹出精简表单）
- 批量 Markdown 导入（`## Q:` / `## A:` 格式）
- 错点记录：记录来源和暴露问题，一键生成易错卡
- 卡片列表批量操作：多选删除、修改校对状态、移动知识点
- 卡片重复检测：编辑时自动提示相似卡片
- Anki TSV 导出（支持 HTML 渲染格式，公式代码直接可见）
- Markdown 文档导出（按知识点分组的 Q/A 格式）
- Anki TSV 增量导入（自动按内容去重）

### 体验优化

- 深色/浅色主题切换，支持跟随系统
- PWA 支持，可安装到桌面，离线可用
- 持久存储保护，浏览器不会自动回收数据
- 考试倒计时、每日目标、学习连续天数
- 移动端响应式布局，汉堡菜单导航
- JSON 导出/导入，导入前自动备份

## 项目边界

- 不是题库，不内置商业 PDF、辅导书原文或课程资料
- 不提供真题解析库
- 本地优先，云同步为可选功能（需注册 Supabase 账号）

## 配套移动端

本项目有配套的 React Native（Expo）移动端应用，共享同一个 Supabase 后端，数据双向同步。移动端代码位于同级目录 `recall-app/`。

- Dexie（IndexedDB）本地持久化
- Supabase（PostgreSQL + Auth）云同步

## 数据说明

用户数据保存在当前浏览器的 IndexedDB 中，应用启动时会请求持久存储权限，防止浏览器自动回收数据。清理浏览器数据仍可能导致卡片和复习记录丢失，请定期导出备份。

支持三种导出方式：
- **JSON 备份**：设置页导出完整数据快照，可用于恢复（含错题本数据）
- **Anki TSV**：导出到 Anki，支持 HTML 格式（公式/代码可见）
- **Markdown**：按知识点分组导出，适合打印或分享

JSON 导入采用覆盖模式：导入前会清空当前本地数据，然后写入备份文件中的数据。Anki TSV 导入为增量模式，自动跳过已有相同内容的卡片。
