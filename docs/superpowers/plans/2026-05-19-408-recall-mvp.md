# 408 Recall MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the 408 Recall MVP so a user can create cards, review due cards with active recall, persist review results, and back up/restore data as JSON.

**Architecture:** A Vue 3 + TypeScript + Vite single-page app stores all user data in IndexedDB through Dexie repositories. Pages use Pinia stores for orchestration, while scheduling, dates, import, and export live in focused services.

**Tech Stack:** Vue 3, TypeScript, Vite, Vue Router, Pinia, Dexie, native CSS.

---

## Scope Decisions

MVP includes knowledge tree display/editing, card CRUD, today's review, review records, and JSON export/overwrite import. MVP excludes AI, mistakes, Markdown/Anki export, accounts, backend services, and synchronization.

Implementation-specific decisions:

- JSON import supports overwrite only.
- Card deletion physically removes the card, its review state, and its review records.
- `archived` is removed from the MVP data model to avoid a second deletion path.
- All local dates use `src/services/dateService.ts`.
- Seed data initialization is guarded by `app_meta.seed_initialized_v1`.
- GitHub Pages deployment must configure Vite `base` from `VITE_BASE_PATH` or default to `/`.
