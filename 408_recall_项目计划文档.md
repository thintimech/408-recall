# 408 Recall 项目计划文档

## 一、项目概述

### 1.1 项目名称

项目暂定名为 **408 Recall**，中文名为 **408 知识记忆助手**。

### 1.2 项目定位

408 Recall 是一个面向计算机考研 408 的知识点记忆与主动回忆工具。项目不以构建完整题库、真题解析库或资料合集为目标，而是帮助用户围绕数据结构、计算机组成原理、操作系统和计算机网络四门课程，建立知识点框架，创建记忆卡片，并通过间隔复习和主动回忆强化长期记忆。

本项目更关注“知识点是否真正记住”和“薄弱内容是否被持续复习”，而不是单纯记录刷题数量。真题和纸质资料仍由用户在线下完成，软件主要负责记录真题暴露出的知识漏洞，并将其转化为可复习的记忆卡片。

### 1.3 项目初衷

408 考研内容覆盖范围广，知识点数量多，很多知识并非完全不会，而是容易出现以下问题：

- 看书时觉得理解，做题时想不起来；
- 概念之间容易混淆，例如进程与线程、分页与分段、TCP 与 UDP；
- 做过的错题没有形成长期复习材料；
- 二战复习中需要反复查漏补缺，但缺少系统化记忆工具；
- 现有资料多为 PDF、笔记或题库，不一定适合主动回忆和长期复习。

因此，本项目希望提供一个轻量、可用、开源友好的 408 记忆工具，让用户可以边复习边沉淀自己的知识卡片。

## 二、项目目标

### 2.1 总体目标

构建一个本地优先、开源友好、支持 AI 辅助制卡的 408 知识点记忆工具，帮助用户完成以下学习闭环：

> 知识点整理 → 创建记忆卡片 → 主动回忆 → 记录掌握情况 → 自动安排复习 → 根据错点补充卡片 → 持续强化记忆

### 2.2 第一版目标

第一版以“能真正用于个人复习”为目标，重点完成以下能力：

1. 内置 408 四门课的基础知识点框架；
2. 支持用户手动创建、编辑和删除记忆卡片；
3. 支持今日复习队列；
4. 支持主动回忆模式；
5. 支持记录复习结果，包括“忘了、困难、记得、熟练”；
6. 根据复习结果自动安排下一次复习时间；
7. 支持错点记录，并将错点转化为记忆卡；
8. 支持 DeepSeek API 辅助生成记忆卡片；
9. 支持 JSON 数据导入导出，方便备份和迁移；
10. 支持部署到 GitHub Pages，便于其他用户访问和使用。

### 2.3 非目标范围

第一版明确不做以下内容：

- 不做完整题库；
- 不做在线模拟考试；
- 不收录商业 PDF、辅导书原文或课程资料；
- 不提供真题解析库；
- 不做用户登录系统；
- 不做多人社区、排行榜和打卡广场；
- 不做支付或商业功能；
- 不做复杂后端和数据库服务。

## 三、目标用户

### 3.1 核心用户

本项目主要面向准备计算机考研 408 的学生，尤其适合已经完成一轮学习、需要强化记忆和查漏补缺的考生。

### 3.2 典型用户场景

1. **二战考生**  
   已经接触过 408 内容，但很多知识点记忆不牢，需要系统复习。

2. **一轮复习结束的考生**  
   知识体系大致建立，但需要把零散知识点转化为长期记忆。

3. **真题复盘用户**  
   做纸质真题后，希望记录暴露出的知识漏洞，并生成后续复习材料。

4. **喜欢主动回忆的用户**  
   不满足于反复看笔记，希望通过问答卡片检查自己是否真正掌握。

5. **希望使用 AI 辅助整理内容的用户**  
   希望借助 DeepSeek 等模型，把知识点、错点和笔记快速转化为记忆卡片。

## 四、核心功能规划

### 4.1 知识点树

系统内置 408 四门课的基础章节框架：

- 数据结构；
- 计算机组成原理；
- 操作系统；
- 计算机网络。

第一版先做到章节级和部分常见知识点级，不追求一次性覆盖所有细节。用户可以自行新增、修改或删除知识点节点。

示例结构：

```text
408
├── 数据结构
│   ├── 绪论
│   ├── 线性表
│   ├── 栈和队列
│   ├── 树与二叉树
│   ├── 图
│   ├── 查找
│   └── 排序
├── 计算机组成原理
│   ├── 计算机系统概述
│   ├── 数据的表示和运算
│   ├── 存储系统
│   ├── 指令系统
│   ├── 中央处理器
│   ├── 总线
│   └── 输入输出系统
├── 操作系统
│   ├── 操作系统概述
│   ├── 进程与线程
│   ├── 处理机调度
│   ├── 同步与互斥
│   ├── 死锁
│   ├── 内存管理
│   ├── 文件管理
│   └── I/O 管理
└── 计算机网络
    ├── 网络体系结构
    ├── 物理层
    ├── 数据链路层
    ├── 网络层
    ├── 传输层
    └── 应用层
```

### 4.2 记忆卡片管理

记忆卡片是系统核心对象。每张卡片对应一个知识点或一个易错内容。

卡片基本信息包括：

- 所属科目；
- 所属知识点；
- 卡片类型；
- 正面问题；
- 背面答案；
- 补充说明；
- 标签；
- 来源类型；
- 校对状态；
- 创建时间；
- 更新时间。

卡片类型包括：

- 概念卡；
- 对比卡；
- 填空卡；
- 流程卡；
- 公式卡；
- 易错卡；
- 大题模板卡。

示例卡片：

```text
类型：对比卡
正面：进程和线程的区别是什么？
背面：进程是资源分配的基本单位，线程是处理机调度的基本单位。同一进程内线程共享进程资源，但每个线程有自己的栈、程序计数器和寄存器状态。
标签：操作系统、进程、线程、易混淆
```

### 4.3 主动回忆模式

复习时，系统先展示卡片正面，用户先自行回忆答案，再点击显示背面内容。

显示答案后，用户根据自己的掌握情况选择：

- 忘了；
- 困难；
- 记得；
- 熟练。

系统根据用户选择更新卡片复习状态，并计算下一次复习日期。

第一版可采用简单复习规则：

| 复习结果 | 下次复习时间 |
|---|---|
| 忘了 | 明天 |
| 困难 | 2 天后 |
| 记得 | 4 天后 |
| 熟练 | 7 天后 |

后续版本可以逐步优化为更接近 Anki 的间隔重复算法。

### 4.4 今日复习

首页展示当天需要复习的卡片数量和分类情况。

示例：

```text
今日待复习：36 张
数据结构：8 张
计算机组成原理：12 张
操作系统：9 张
计算机网络：7 张
```

复习完成后，首页展示当日进度：

```text
已完成：20 / 36
忘了：5 张
困难：8 张
记得：5 张
熟练：2 张
```

该模块的目标是让用户每天打开软件后，能够直接知道今天该复习什么。

### 4.5 错点记录

错点记录用于承接纸质真题、刷题本或平时练习中暴露的问题。

系统不保存完整真题题干和解析，只记录题目来源和暴露出的知识问题。

示例：

```text
来源：2021 年 408 真题第 37 题
暴露问题：Dijkstra 算法过程不熟，无法准确写出每一步最短路径更新结果。
关联知识点：数据结构 / 图 / 最短路径
处理方式：生成一张易错卡并加入复习队列
```

错点处理流程：

```text
做纸质真题
↓
发现知识漏洞
↓
记录错点
↓
关联知识点
↓
生成记忆卡
↓
加入复习队列
```

### 4.6 DeepSeek AI 辅助制卡

系统支持用户在设置中填写 DeepSeek API Key。AI 功能为可选项，不影响基础功能使用。

AI 主要用于：

1. 根据知识点生成记忆卡；
2. 根据错点生成易错卡；
3. 根据长笔记生成问答卡；
4. 生成概念对比卡；
5. 生成今日复习建议。

AI 生成内容默认标记为“未校对”，用户确认后才能改为“已校对”。

AI 使用原则：

- AI 是辅助整理工具，不是权威答案来源；
- AI 生成内容需要人工检查；
- 项目不把 AI 生成内容直接当作标准资料；
- 用户可以选择不开启 AI 功能。

### 4.7 数据导入导出

为了保证用户数据安全和开源可用性，第一版支持：

- 导出全部数据为 JSON；
- 从 JSON 恢复数据；
- 导出卡片为 Markdown。

后续可考虑：

- Anki 格式导出；
- CSV 导入；
- Markdown 批量制卡；
- 多设备同步。

## 五、技术方案

### 5.1 技术选型

第一版采用纯前端本地应用方案：

| 模块 | 技术 |
|---|---|
| 前端框架 | Vue 3 |
| 开发语言 | TypeScript |
| 构建工具 | Vite |
| UI 组件 | Naive UI 或 Element Plus |
| 状态管理 | Pinia |
| 路由管理 | Vue Router |
| 本地数据库 | IndexedDB |
| AI 接口 | DeepSeek API |
| 部署方式 | GitHub Pages |

### 5.2 不使用后端的原因

第一版暂不使用 Spring Boot、MySQL 和用户系统，原因如下：

1. 项目目标是个人学习工具，不需要复杂服务端；
2. 纯前端更容易部署到 GitHub Pages；
3. 其他用户使用门槛更低，不需要配置数据库；
4. 数据保存在本地，更适合个人知识卡片；
5. 后续如果确实需要同步功能，再扩展后端服务。

### 5.3 本地优先原则

项目采用“本地优先”设计：

- 用户数据默认保存在浏览器 IndexedDB；
- 用户可以随时导出 JSON 备份；
- AI 功能可选，不影响离线使用；
- 项目不强制注册账号；
- 用户对自己的学习数据拥有完整控制权。

## 十、风险与应对

### 10.1 数据来源风险

风险：合适的 408 开源结构化资料较少，直接搬运 PDF、题库或商业资料存在版权风险。

应对：项目不内置商业题库和解析，只提供通用知识点框架和制卡工具。具体卡片由用户自行创建，项目示例卡片尽量使用自写内容。

### 10.2 AI 内容准确性风险

风险：AI 生成内容可能存在错误，尤其是 408 中涉及算法、计组计算、协议细节等内容。

应对：AI 生成内容默认标记为未校对，用户确认后才能标记为已校对。界面中提示 AI 内容仅供辅助学习。

### 10.3 本地数据丢失风险

风险：浏览器缓存清理可能导致 IndexedDB 数据丢失。

应对：提供明显的数据导出功能，并在首页或设置页提醒用户定期备份 JSON 文件。

### 10.4 项目范围膨胀风险

风险：后续可能想加入题库、社区、考试、账号系统等功能，导致项目复杂度过高。

应对：V1.0 前坚持本地记忆工具定位，不做平台化功能。

## 十一、开源说明草案

本项目开源时应明确说明：

1. 项目不内置商业题库；
2. 项目不收录辅导书原文；
3. 项目不提供真题解析；
4. 项目主要提供知识记忆、主动回忆和间隔复习能力；
5. 用户应自行确保导入内容的合法性；
6. AI 生成内容需要用户自行校对。

README 可使用如下表述：

> 408 Recall 不是题库，也不是资料合集。它是一个面向计算机考研 408 的知识点记忆与主动回忆工具，帮助用户把数据结构、计算机组成原理、操作系统和计算机网络中的知识点拆成可复习、可回忆、可追踪的记忆卡片。本项目不内置商业题库，不收录辅导书原文，不提供真题解析。用户可以基于自己的笔记、真题复盘和学习资料创建个人卡片，并通过间隔复习强化记忆。

## 十二、当前结论

408 Recall 第一版应坚持以下方向：

- 做知识记忆工具，不做题库平台；
- 做主动回忆，不做单纯笔记堆叠；
- 做本地优先，不做复杂后端；
- 做 AI 辅助制卡，不让 AI 替代人工判断；
- 做开源友好，不内置版权风险资料；
- 做真正能每天使用的工具，而不是只看起来完整的平台。

第一版最重要的判断标准是：

> 用户能否每天打开它，复习到期卡片，记录遗忘内容，并把真题暴露出的问题转化为后续可复习的知识卡片。

只要这个闭环跑通，项目就已经具备实际价值。
